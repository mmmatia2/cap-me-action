# Architecture Decision Records

Use ADRs for decisions that affect architecture, contracts, or long-term maintenance.

## Naming

- File format: `NNNN-short-title.md`
- Example: `0001-sync-backend-strategy.md`

## When To Create An ADR

- New sync/storage strategy
- Data model/schema evolution
- Security/privacy tradeoff
- UI architecture changes that impact maintainability

## Template

Copy `docs/adr/0000-template.md` for new ADRs.
